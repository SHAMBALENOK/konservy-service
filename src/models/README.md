# Models Layer (Data & Schemas)

Слой данных и схем. Определяет структуру хранения и передачи информации.

## 🎯 Ответственность
- **SQLAlchemy Models**: Маппинг Python-классов на таблицы БД.
- **Pydantic Schemas**:
  - `RequestSchemas`: Валидация входящих данных.
  - `ResponseSchemas`: Структура исходящих JSON.
  - `DomainDTOs**: Внутренние объекты передачи данных между слоями.

## ⛔ Чего здесь НЕ должно быть
- Бизнес-методов (никаких `def calculate_balance()` внутри моделей БД).
- Логики сохранения данных (save/update методы вынесены в репозитории/сервисы).

## 📝 Правила
1. Модели БД находятся в `db_models.py` (или подпапке `db/`).
2. Pydantic схемы находятся в `schemas.py` (или подпапке `api/`).
3. Используйте `ConfigDict` в Pydantic для настройки сериализации (например, скрытие полей).
4. Любые изменения структуры БД требуют новой миграции Alembic.
