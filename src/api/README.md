# API Layer (Presentation)

Этот слой отвечает за взаимодействие с внешним миром (HTTP/WebSocket).

## 🎯 Ответственность
- Прием HTTP-запросов и валидация входных данных через **Pydantic схемы**.
- Маршрутизация запросов к бизнес-логике (`core/`).
- Форматирование ответов (стандартизация JSON-структур, коды ошибок).
- Middleware: аутентификация (JWT), rate-limiting, логирование запросов.

## ⛔ Чего здесь НЕ должно быть
- Бизнес-логики (расчеты, проверки балансов, лимиты).
- Прямых обращений к базе данных или ORM.
- Сложных преобразований данных (маппинг должен быть в `core/` или `models/`).

## 📝 Правила
1. Все эндпоинты должны иметь аннотации типов.
2. Используйте зависимости FastAPI (`Depends`) для получения сервисов из `core/`.
3. Ошибки должны возвращаться в едином формате (см. `exceptions.py`).
4. sensitive data (пароли, токены) никогда не логируются здесь в открытом виде.
