# Core Layer (Business Logic)

Сердце приложения. Здесь живут банковские правила, сценарии использования (Use Cases) и управление транзакциями.

## 🎯 Ответственность
- Реализация бизнес-сценариев: `create_account`, `transfer_funds`, `calculate_interest`.
- Валидация бизнес-правил (достаточно ли средств? активен ли счет?).
- Управление атомарностью операций (начало/коммит/откат транзакций БД).
- Генерация событий домена (если используется событийная модель).

## ⛔ Чего здесь НЕ должно быть
- Знаний о HTTP (request/response objects).
- Прямого импорта SQLAlchemy моделей для маппинга таблиц (используйте абстракции или DTO).
- Конфигурации внешних сервисов (API ключи, URL).

## 📝 Правила
1. Каждая функция/метод должна быть идемпотентной или иметь механизм защиты от дублей.
2. Транзакции управляются явно: `async with session.begin(): ...`.
3. Логика должна быть тестируемой без поднятия веб-сервера (Unit Tests).
4. Исключения должны быть специфичными для домена (например, `InsufficientFundsError`).
