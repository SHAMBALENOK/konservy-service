# Configuration

Управление настройками приложения для разных окружений (Dev, Stage, Prod).

## 🎯 Ответственность
- Загрузка переменных окружения (`.env`).
- Валидация настроек при старте приложения (через Pydantic Settings).
- Хранение констант и feature-flags.

## 📂 Структура
- `settings.py`: Основной класс настроек.
- `.env.example`: Шаблон файла переменных окружения (без реальных секретов!).

## 📝 Правила
1. Никогда не коммитьте реальный `.env` файл в Git.
2. Все секреты (DB_PASSWORD, SECRET_KEY) должны браться только из env vars.
3. Если добавляете новую настройку, обновите `.env.example` и документацию.
4. Используйте типизацию для всех настроек.
